# Telegram Instant View

Ссылки на посты блога открываются в Telegram как Instant View — мгновенная
«читалка» с кнопкой под превью, без загрузки сайта.

## Как это работает

Когда кто-то делится ссылкой, серверы Telegram запрашивают страницу
краулером с User-Agent `TelegramBot (like TwitterBot)`. SSR-сервер
(`front/server.ts`) узнаёт его по этой строке и вместо обычной страницы
отдаёт минимальную разметку из `front/server/instantview.ts` — обычные
читатели и поисковики её никогда не видят.

Официальный путь к Instant View — шаблон, одобренный командой Telegram, —
фактически закрыт: очередь ревью мертва с 2019 года. Поэтому используется
известный обходной приём: страница мимикрирует под
[Teletype](https://teletype.in), у которого есть одобренный шаблон,
срабатывающий не по домену, а по признакам самой страницы:

- мета-тег `tg:site_verification` (значение скопировано с продовых страниц
  Teletype — своё сгенерировать нельзя, механизм недокументирован);
- вёрстка Teletype: `div.site-content.article > article.article__content >
  div.entry-content`, а для видео — точная структура
  `figure.m_column > div.wrap > video.embed-viedo` (опечатка «viedo» — из
  прода Teletype, она обязана совпадать).

Тот же приём использует WordPress-плагин
[tg-instantview](https://github.com/petrows/wp-tg-instantview).

## Как проверить

- **Разметка глазами:** `https://repin.pw/posts/<slug>?tg-iv=1` — тот же
  рендер, что видит краулер, без подмены User-Agent.
- **Сброс кэша Telegram:** превью и результат разбора кэшируются на стороне
  Telegram (в том числе неудачный!). После правок отправьте ссылку боту
  [@WebpageBot](https://t.me/WebpageBot) — он перечитает страницу.

## Ограничения и риски

- Механизм недокументирован: если Telegram изменит проверку, Instant View
  молча пропадёт. Деградация безопасная — останется обычное превью по
  `og:`-тегам.
- Краулер не исполняет JavaScript — всё должно быть в первом HTML-ответе
  (для нашего SSR это и так верно).
- Instant View есть не на всех платформах Telegram (например, нет на
  Linux Desktop).
- Если пост не найден или API недоступен, запрос прозрачно проваливается
  в обычный SSR — хуже обычного превью не бывает.
