# Наполнение блога из Telegram (`sync`)

Шаг необязательный. Блог поднимается и пустым — команда `sync` нужна, когда
хочется залить в него посты из своего Telegram-канала

`sync` заходит в Telegram под вашим личным аккаунтом (не бот), читает историю
канала, нормализует сообщения в посты и раскладывает их: текст — в Postgres,
медиа — в MinIO. Это разовая команда, а не постоянно работающий сервис

## 1. Зарегистрировать приложение Telegram

Нужны `api_id` и `api_hash` — ключи доступа к пользовательскому API Telegram

1. Откройте <https://my.telegram.org> и войдите по номеру телефона (код придёт
   в приложение Telegram, не по SMS)
2. Раздел **API development tools**
3. **Create application**, заполните поля:
   - **App title** — любое, например `repin.pw`
   - **Short name** — например `repinpw`
   - **Platform** — *Desktop* (или *Other*)
   - URL и описание можно оставить пустыми
4. В ответ получите **App api_id** (число) и **App api_hash** (строка).
   Это секреты — храните их как пароль

## 2. Заполнить `.env`

В корневом `.env` (для запуска через Docker) или в `api/.env` (для локального
запуска без Docker):

```dotenv
TELEGRAM_API_ID=1234567
TELEGRAM_API_HASH=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TELEGRAM_PHONE=+79991234567     # номер аккаунта, под которым читаем канал
TELEGRAM_CHANNEL=allrpn         # username канала без @
```

Аккаунт должен иметь доступ к каналу: для приватного — состоять в нём

## 3. Запустить

Через Docker (стек уже поднят через `make up`):

```bash
make sync
```

Локально:

```bash
cd api
make sync            # = go build + ./bin/sync
```

## 4. Первый вход — интерактивный

Телефон берётся из `TELEGRAM_PHONE`, а код подтверждения нужно ввести вручную:

```
Enter code: 12345
```

Если на аккаунте включена двухфакторная защита — попросит облачный пароль:

```
Enter 2FA password: ********
```

После входа сессия сохраняется, и повторная авторизация не нужна:

- Docker — `.docker/data/telegram/session.json`
- локально — `api/var/telegram/session.json`

Файл сессии — это фактически доступ к аккаунту. Не коммитьте его и не
передавайте (он уже под `.gitignore`)

## 5. Повторный запуск и пересборка

- **`make sync`** можно запускать снова — подтянет новые сообщения канала
- **`make rerender`** (`./bin/cli rerender`) — пересобрать заголовки и текст
  постов из уже сохранённых сообщений, без обращения к Telegram. Удобно после
  правок логики нормализации

## Возможные проблемы

| Симптом | Причина и что делать |
| --- | --- |
| `FLOOD_WAIT_x` | Слишком частые запросы. Подождите указанные `x` секунд и повторите |
| `CHANNEL_PRIVATE` / нет доступа | Аккаунт не состоит в канале или неверный `TELEGRAM_CHANNEL` |
| Код не приходит в SMS | Telegram присылает код в само приложение, а не по SMS |
| Нужно сбросить вход | Удалите файл `session.json` и запустите `sync` заново |
